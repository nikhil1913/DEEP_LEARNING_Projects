Deep Learning Project Brief

An in-depth guide to developing the EcoSort AI Waste Classifier.

By Nikhil Singh Bisht

Project Overview

This project is a deep learning-based image classification system designed to identify different types of waste from images. It uses TensorFlow and Keras to build and train models, starting with a custom CNN and then exponentially improving performance using transfer learning with MobileNetV2.

The core objective is to automate the categorization of 20 unique waste types, aiding in the global effort toward more efficient recycling and waste management.

1. Data Acquisition & Sanitization

Developing a high-accuracy classifier requires a diverse and balanced dataset. We built a custom image scraper using the icrawler library to fetch images from Google and Bing.

  • Categories: 20 distinct classes (e.g., Plastic bottles, Cardboard, Styrofoam, Bio-medical waste).
  • Volume: Approximately 250 images per category were scraped.

After scraping, a rigorous Data Cleaning process was initiated. We implemented a script to check the integrity of every image file. Using tf.image.decode_image, we identified and removed corrupted files that could crash the training process, ensuring only valid RGB images remained.

2. Preprocessing & Augmentation

Before training, images were resized to 222x222 pixels. To enhance the model's ability to generalize to new, unseen photos, we applied heavy Data Augmentation:

Rotation & Zoom
Horizontal & Vertical Flips
Translation Shifts

Normalization was critical—input pixel values were rescaled to the range [-1, 1], aligning with the expectations of the pre-trained weights.

3. The Custom CNN (Initial Failure)

Every breakthrough follows a challenge. We initially designed a custom Convolutional Neural Network (CNN) from scratch. Despite careful tuning, this model struggled with the high variance of the 20 waste categories.

Result: The custom CNN peaked at a mere 19% accuracy. Recognizing that the model was not converging, the training cell was stopped. Building a model from scratch for such complex visual features proved inefficient compared to leveraging pre-learned patterns.

4. MobileNetV2 & Transfer Learning

We pivoted to Transfer Learning using MobileNetV2, a state-of-the-art architecture optimized for performance. By using weights pre-trained on the ImageNet dataset, the model inherited a deep understanding of shapes, colors, and textures.

Model Architecture

base_model = MobileNetV2(
    input_shape=(224, 224, 3),
    include_top=False,
    weights='imagenet'
)
base_model.trainable = False

model = Sequential([
    base_model,
    GlobalAveragePooling2D(),
    Dense(128, activation='relu'),
    Dropout(0.4),
    Dense(64, activation='relu'),
    Dense(20, activation='softmax')
])

5. Fine-Tuning & Deployment

After reaching a stable accuracy with frozen weights, we initiated Fine-Tuning. We un-froze the last 2 convolutional layers of the MobileNetV2 base and re-trained the model with a very low learning rate (1e-5).

This allowed the model to adjust its deepest filters to the specific visual nuances of trash and recyclables, eventually reaching over 73% validation accuracy.

The final trained model is saved in .keras format and deployed via this FastAPI-powered web application for real-time inference.
